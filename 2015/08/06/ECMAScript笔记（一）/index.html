ECMAScript笔记（一）

发表于 2015-08-06

一致性，对标准的ECMAScript实现

  • 必须支持规范所描述的 [ 类型、值、对象、属性、函数、程序语法、语义 ]
  • 能解析Unicode 3.0 或更高版本
  • 允许提供除规范描述的 [ 类型、值、对象、属性、函数 ]
  • 允许支持规范描述的程序语法和正则表达式语法

语言概述

  • ECMAScript基于对象，基本设施由对象提供。
  • ECMAScript对象是[ 属性 ]的集合，每个属性有 0 或 多个 [ 特性 ]
  • 函数是可调用对象，方法是通过属性和对象关联的函数
  • 属性是持有其他对象、原始值、函数的容器
  • 原始值：Undefined，Null，Bollean，Number，String，Object
  • ECMAScript内置对象：全局对象，Object，Function，Array，JSON,String，Boolean，Number，Math，Error，RegExp，EvalError，RangeError，ReferenceError，TypeError，URIError，SyntaxError
  • ECMAScript内置运算符：一元、乘法、加法、剪发、按位移、关系、相等、二进制位、二进制逻辑、赋值，逗号运算符

ECMAScript对象

  • ECMAScript不使用C++/JAVA中的类，通过 [ 字面量 ] 或 [ 构造器 ] 创建对象
  • 构造器是拥有名为 [ prototype ] 属性的函数
  • [ prototype ] 用于实现 [ 原型继承 ] 和 [ 共享属性 ]
  • 构造器通过 [ new 表达式 ] 创建对象
  • 由构造器创建的对象，都有一个隐式引用 [ 对象的原型 ] 链接到构造器的 [ prototype ]
  • [ 原型链 ]是每个构造器的 [ prototype ] 对象可能有一个非空的 [ 对象的原型 ] 链接到另外构造器的[ prototype ] 组成的一条 [ 一定规则对象链 ]
  • 相对基于类面向对象的实例拥有状态，类拥有方法，并只能继承结构和行为，ECMAScript对象则全部可拥有或继承
